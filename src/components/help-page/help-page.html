CoReBIN Comparison and Evaluation of metagenomic binning

Introduction

CoReBIN is a user-friendly web-based tool to compare the results of different metagenomic binning methods and to aid manual refinement of the bins. The binning result of two different methods can be visually compared in a chord diagram. Individual bins can be inspected and refined using GC%, coverage and tetranucleotide frequencies.

User interface

  1. Home: Introduction and upload of metagenomic assemblies and bin-sets.
  2. Overview: Summary of the selected (selection using dropdown in 6) metagenomic assembly (contig lengths, GC% of contigs). In addition, the characteristics of the uploaded bin-sets can be inspected here.
  3. Compare: Visually compare two different binning results.
  4. Refine: Visual refinement of bins using GC%, coverage, tetranucleotide frequencies.
  5. Help: Documentation on usage, file formats and demo data.
  6. Assembly selection: Select your metagenomic assembly of interest (upload in Home-panel).
  7. Binset selection: Select your binset assembly of interest (upload in Home-panel).

Uploading your data

Users can upload their own data in the Home-panel. The following data is needed to analyse your metagenomic bins using CoReBIN:

  • Metagenomic assembly
  • Coverage data not required
  • Metagenomic bins
    • (i.e. metagenomic binning results)
First upload your metagenomic assembly and corresponding coverage data (if available) in the "1. Upload assembly"-box (this might take a while, be patient!) and subsequently upload your metagenomic bin set in the "2. Upload bin set".-box
Details on file format are given below.

1. Metagenomic assembly and Coverage data

Metagenomic assembly

Fasta format. The headers have to correspond to the contig IDs used in the coverage table and the contig IDs used in the binning table(s). We have limited the upload of metagenomic assemblies to files up to xxx mb, larger metagenomic assemblies can not be analyzed using CoReBIN.

Coverage data

You can add a file containing coverage information when uploading a metagenomic assembly. Sequencing coverage describes the average number of reads that align to, or "cover," your metagenomic assembly. You can determine the coverage by mapping your reads to your assembly using tools such as BWA and Bowtie2. Differential coverage information is utilized by many metagenomic binning methods and can be very useful to refine bins.

The input file should be a tab separated file with on each line a contigID followed by the coverage in one or more samples, i.e. if you have coverage data on one sample you have one additional column, if you have two different coverages you will need two additional columns.

ContigID1 <tab> Coverage_sample1 <tab> Coverage_sample2
ContigID2 <tab> Coverage_sample1 <tab> Coverage_sample2
ContigID3 <tab> Coverage_sample1 <tab> Coverage_sample2
...<tab> ... <tab> ...
...<tab> ... <tab> ...

It is important that the contig IDs are exact matches with the ones in the fasta headers of the assembly.

2. Metagenomic bins

ContigID1 <tab> BinId
ContigID4 <tab> BinId
ContigID2 <tab> BinId
...<tab> ...
...<tab> ...

Again, it is important that the contig IDs are exact matches with the ones in the assembly. Unbinned contigs can be left out in this table and no specific ordering is required.

Metagenomic binning

Recently published metagenomic binning programs and methods:

  • CONCOCT: Alneberg, Johannes, et al. "Binning metagenomic contigs by coverage and composition." Nature methods 11.11 (2014): 1144-1146.
  • MetaBAT: Kang, Dongwan D., et al. "MetaBAT, an efficient tool for accurately reconstructing single genomes from complex microbial communities." PeerJ 3 (2015): e1165.
  • GroopM: Imelfort, Michael, et al. "GroopM: an automated tool for the recovery of population genomes from related metagenomes." PeerJ 2 (2014): e603.
  • MaxBin 2.0: Wu, Yu-Wei, Blake A. Simmons, and Steven W. Singer. "MaxBin 2.0: an automated binning algorithm to recover genomes from multiple metagenomic datasets." Bioinformatics (2015): btv638.
  • VizBin: Laczny, Cedric C., et al. "VizBin-an application for reference-independent visualization and human-augmented binning of metagenomic data." Microbiome 3.1 (2015): 1.
  • COCACOLA: Lu, Yang Young, et al. "COCACOLA: binning metagenomic contigs using sequence COmposition, read CoverAge, CO-alignment, and paired-end read LinkAge." Bioinformatics (2016): btw290.
Most of these methods use differential coverage information. All programs will return a fasta file for each bin that they produce.

Demo data-set

A simulated dataset of short sequences consisting of three different samples, each containing 10.000.000 paired-end 100 nt reads, was created using the supplied error model for Illumina GA IIx with TrueSeq SBS Kit v5-GA using GemSim (McElroy, Luciani & Thomas, 2012). Reads were derived from a set of 10 reference genomes and chosen with probability proportional to the species abundance and genome size (see table below).

Subsequently, the simulated reads were co-assembled using MEGAHIT (Li et al, 2015). Coverage for the three different samples was determined using bowtie2 (Langmead & Salzberg, 2012). The assembly and differential coverage table were then used as input for two different binning programs (default parameters): MaxBin 2.0 (Wu, Simmons and Singer, 2015) and MetaBAT (Kang et al, 2015).
